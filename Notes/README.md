<h1>Overall Notes/Though Process</h1>

<p>
This folder contains the structured notes, planning documentation, and reflective thinking process used throughout the design, implementation, and troubleshooting of this pfSense-based homelab network project.
</p>

<p>
Rather than focusing solely on final configurations, this documentation captures the reasoning behind each decision, the challenges encountered during development, and the iterative process used to refine the network architecture over time.
</p>

<p>
The purpose of this README is to provide context for how the system evolved from an initial basic firewall setup into a fully segmented, multi-VLAN infrastructure with centralized DNS, secure remote access, and hardened firewall policies.
</p>

<p>
Throughout the project, multiple design decisions were evaluated and adjusted based on real-world behavior rather than theoretical planning alone. This includes troubleshooting NAT issues, refining VLAN trunking between pfSense and the core switch, and restructuring firewall rules after observing unexpected traffic behavior.
</p>

<p>
These notes also reflect the learning process involved in working with enterprise-style networking concepts such as VLAN segmentation, DHCP scope isolation, DNS centralization, and secure remote administration using overlay networking solutions like Tailscale.
</p>

<p>
Overall, this folder serves as both a technical journal and a problem-solving log, documenting not just what was built, but why specific choices were made and how the final architecture was achieved through iterative improvement and testing.
</p>
